## 3. 南京大学冬令营：LaViRA 复现与 GO2 初步部署

接下来这一部分，是我在南京大学具身智能 winter camp 中参与的工作。我主要参与了 **LaViRA** 框架的复现、调试与改进，并完成了从仿真环境到四足机器人 GO2 平台的初步部署探索；在小组中，我负责的是多模态模型调用与导航链路分析，以及系统稳定性相关的一些改进。

### 3.1 LaViRA 这篇论文在做什么

我对 LaViRA 的理解是，它不是一个单纯“输入图像和语言，直接端到端输出动作”的黑箱式 VLN 模型，而是一个更偏**系统化、分层化**的具身导航框架。
它要解决的任务是：输入 RGBD 图像流和自然语言指令，让机器人在未知环境里逐步导航到目标位置，并在合适的时候执行 STOP。这个任务的难点在于，它同时涉及语言理解、视觉感知、空间定位、路径规划和真实执行，链路比较长，而且每一层都可能出错。

因此，LaViRA 采用了一种比较自然的思路：**把复杂问题拆开做**。
在我自己的理解里，我会把它概括成三层，也就是 **LA、VA、RA**：

* **LA：Language Action**
  这一层负责“基于语言指令做高层语义决策”。也就是说，机器人不是上来就直接生成底层动作，而是先回答“下一步应该朝什么语义目标推进”“当前任务阶段是什么”“是否已经满足停止条件”。它更像是一个高层任务规划器。

* **VA：Vision Action**
  这一层负责“把高层语言目标和当前视觉环境对齐起来”。比如当前场景里有哪些物体、哪些区域与指令描述相关、目标可能落在哪个 bbox 或哪个语义区域。这里会结合视觉模型、开放词汇检测以及全景观察来减少单视角盲区。它本质上是在回答：“语言里说的目标，当前视觉上应该怎么落地定位？”

* **RA：Robot Action**
  这一层负责“把已经确定的目标转成机器人真正能执行的动作和路径”。比如目标区域怎么投影到世界坐标、如何利用地图和 FMM 生成可执行路径、机器人如何一步步走过去。它对应的是从高层意图到底层控制的最后一公里。

如果把它放回到整个 VLN 工作流里看，其实对应关系会很清楚：

* **语言输入进来之后，先由 LA 做高层理解和阶段决策**
* **再由 VA 结合视觉感知和环境观测完成目标定位**
* **最后由 RA 调用地图与规划模块，把目标转化为具体导航动作**

也就是说，它把“听懂指令”“看懂环境”“真正走过去”这三件事拆开了。
我觉得这种拆法非常有意义，因为 VLN 本来就是一个多环节耦合问题。如果不拆，最后出问题时很难知道到底是语言没理解对、视觉没看准，还是路径规划和执行出了问题；而拆开以后，每一层的职责更清晰，可解释性更强，也更方便替换、调试和部署。

### 3.2 论文里的具体链路

从代码和汇报内容来看，LaViRA 整体可以对应到三大阶段：**Perception、Reasoning、Action**。

首先是 **Perception**，也就是感知层。
这里会用 GroundedSAM 一类的开放词汇目标检测方法去提取语义目标，并逐步构建语义地图。它做的事情本质上是把原始视觉输入变成更适合导航推理的空间语义表示。

接着是 **Reasoning**，也就是推理层。
这里有两个很关键的设计。一个是 **Look Around**，也就是通过全景视角而不是单一前视角来观察环境，减少盲区问题；另一个是 **VLM Planner**，它会结合当前环境状态和语言指令，输出下一步的导航目标，通常表现为一个目标 bbox，或者直接输出 STOP。

最后是 **Action**，也就是执行层。
这里会把目标 bbox 投影到三维世界坐标，再结合 **FMM（Fast Marching Method）** 生成具体路径点，从而驱动机器人完成导航。也就是说，它不是只做抽象语义判断，而是真的把语义目标通过空间映射和规划变成可执行路径。

### 3.3 LaViRA 的创新点

如果概括这篇工作的创新，我觉得主要有三点。

第一，它把 VLN 从“纯 benchmark 方法”往“可部署机器人系统”推进了一步。
它不是只关心预测动作对不对，而是打通了**语言理解—视觉定位—空间投影—路径规划—机器人执行**这条完整链路。

第二，它的**分层设计**很重要。
LA、VA、RA 这种拆分，使得每一层职责清晰，也更方便做模块替换和问题定位。比如在复现里我们就会尝试替换不同的 LA / VA 模型，看看高层语言推理能力和视觉识别能力对最终导航效果的影响。

第三，它的系统更接近真实机器人应用。
通过语义地图、全景观察和 FMM 路径规划，它不是停留在抽象动作预测，而是真正把大模型能力接到了机器人导航执行链里。

### 3.4 我在 LaViRA 复现里做了什么

我在这个项目里主要做的是多模态模型调用与导航链路分析，也就是说，我更关注整个系统是怎么跑起来的，以及链路中哪些地方最容易失稳。

在复现过程中，我们遇到的典型问题包括：

* 模型输出不稳定
* 过早 STOP
* 长距离盲走
* 输出格式崩坏，导致下游解析失败
* 真机闭环时鲁棒性下降

围绕这些问题，我尝试过几类改进：

* 更换 **LA / VA** 模型，提升高层推理和视觉识别的稳定性
* 引入**语义 + 几何双重 STOP** 机制，避免只靠单一条件停止
* 优化目标识别与路径规划逻辑
* 构建 **Fallback / 正则解析** 兜底机制，提高系统对异常输出的容错性

### 3.5 真机部署给我的启发

后面我们还做了 GO2 平台上的初步部署探索。这个过程让我第一次比较直观地体会到，Sim2Real gap 并不是一个抽象概念，而是会具体表现为感知噪声、接口兼容、通信链路稳定性和实际运动扰动等一系列问题。
对我来说，这段经历最大的价值在于：LaViRA 这种分层系统并不意味着它没有误差累积，恰恰相反，它的链路很长；但正因为它拆得足够清楚，所以你更容易知道问题究竟出在语言理解、视觉定位还是路径执行。

## 4. 从 LaViRA 到历史环境地图表示：为什么我开始关注 GridMM

这一页是我现在最想强调的一页。
在复现 LaViRA 的过程中，我逐渐意识到，长程导航中很多问题不只是某个模块“这一帧没看清”或者“这一轮输出不稳”，而是更深层地落在：**机器人到底是怎么记住它已经走过的环境的。**

换句话说，我开始更关注 **历史环境的存储与表示方式**。

如果回看 LaViRA，其实它对历史环境的利用，更多还是一种**基于历史上下文和语义地图的高层决策模式**：
它会通过已有观测逐步构建语义地图，再结合当前指令和全景视觉状态，让高层模块判断下一步该往哪里走、什么时候该停。也就是说，它确实在使用历史信息，但它并没有把“历史环境表示本身”作为最核心的方法创新去展开。

也正因为这样，我开始进一步去想：
对于长程 VLN 来说，历史环境信息究竟应该以什么形式被保留下来，才能更稳定地支撑后续决策？这就把我的兴趣自然引到了 **GridMM** 这类工作上。

## 5. GridMM：历史环境地图表示问题

### 5.1 我怎么理解这条技术路线

围绕历史环境表示这个问题，我现在会把 VLN 里的方法大致分成几类。

第一类是 **RNN 隐状态**。
这种方式相当于“不显式画地图”，只是把历史经历压进 hidden state 里。优点是简单，但长程任务里容易丢信息，也不够可解释。

第二类是 **拓扑图**。
它会显式记录去过哪些关键点、节点之间怎么连通，比隐状态更结构化，也更适合长距离推理；但通常偏稀疏，对细粒度视觉和局部空间细节保留有限。

第三类是 **俯视语义图 / BEV 地图**。
这类方法开始显式构造俯视空间表示，并叠加语义信息，更适合做空间推理和目标定位。

第四类就是 **GridMM** 的可增长网格记忆。
它想做的不是只记几个关键节点，也不是只做比较粗的语义区域，而是构建一个可增长的俯视网格地图，在地图层面尽量保留细粒度视觉线索和空间结构。

### 5.2 GridMM 这篇论文讲了什么

GridMM 关注的核心问题是：
**在长程 VLN 中，历史环境应该怎样被保存和读取，才能真正支撑后续导航决策。**

它的整体思路是，智能体在探索过程中不断接收当前视觉观测，并把这些观测持续写入一张**会增长的俯视网格记忆**里。
这张 grid memory 不是静态地图，而是随着探索范围逐步扩展；在决策时，模型既看这张全局历史地图，也看当前局部候选视角信息，再综合做 action scoring 和 STOP 决策。

### 5.3 GridMM 的创新点

我理解它的创新主要有三点。

第一，它提出了一种**可增长的 grid memory**。
和 RNN hidden state 相比，它更显式；和稀疏拓扑图相比，它又更细粒度。

第二，它强调**全局历史地图和局部当前决策的联合建模**。
也就是说，动作不只是看当前帧，而是真正让历史环境参与后续导航。

第三，它把“地图表示本身”变成方法设计重点。
我觉得这点特别重要，因为这意味着问题不再只是“backbone 用什么”，而是 agent 到底该如何组织走过的环境。这个角度和我现在的兴趣点非常一致。

### 5.4 我在复现 GridMM 时重点关注什么

在实践上，我已经完成了 GridMM 的代码复现，并在 R2R 和 REVERIE 数据集上做了运行和记录。对我来说，重点不是只把结果跑出来，而是拆机制。

我主要关注三个问题：

第一，**global memory 和 local memory 的分工**。
也就是全局地图到底提供什么信息，局部候选视角又在什么地方影响最终决策。

第二，**位置特征是怎么进入 scoring 路径的**。
因为长程导航里，位置编码和历史轨迹怎么参与融合，会直接影响动作判断。

第三，**STOP 为什么容易出问题**。
我越来越觉得，过早 STOP 不是一个孤立 bug，而是“历史环境表示—当前语义理解—动作决策”衔接不稳的外在表现。所以我也开始思考，STOP 是否应该从单一几何判断扩展到语义和几何联合判断。

### 5.5 我目前的延伸思考

在复现和分析 GridMM 之后，我现在在想的下一步，是把这类地图记忆思想往**无人机 VLN** 上迁移。
因为我觉得它研究的本质是“历史环境怎么表示”，这个问题在 UAV 场景里同样存在，甚至会更突出，因为无人机的视角变化更大，空间更开阔，高度信息也更重要。

所以我目前的想法不是一上来直接做完整 3D，而是更倾向于先从 **2D 向 2.5D 的 AirNav 场景**推进：保留俯视地图这个主结构，同时逐步把高度和垂直关系引入表示中。

另外，我也有一个比较具体的延伸设想，就是在 GridMM 的基础上进一步探索**分层或多尺度网格表示**。
原始 GridMM 使用的是单一尺度的网格记忆，虽然对局部细节保留得比较好，但长距离导航中所有信息都停留在同一粒度上，不一定有利于形成更高层次的区域结构理解。
所以我在想，能不能构建类似金字塔的多层 grid memory：低层保留细粒度局部观测，高层聚合更抽象的区域级语义和全局布局；在此基础上，再把空间邻近且语义相似的格子动态聚合成“超级单元”，让模型不仅记住“有哪些格子”，还逐步形成“有哪些区域”的理解。
我觉得这类方向无论对长程导航，还是对以后往 2.5D 无人机场景迁移，都是比较有潜力的。

## 你面试时最值得背熟的两段

### LaViRA 精简版

LaViRA 是一个分层的具身导航系统，我会把它概括成 LA、VA、RA 三层：LA 负责基于语言做高层任务决策，VA 负责把语言目标和当前视觉环境对齐，RA 负责通过地图和 FMM 把目标转成可执行路径。这样拆分的意义在于，把“听懂指令”“看懂环境”“真正走过去”这三件事分开做，系统更可解释、可替换，也更方便调试和部署。它的创新点就在于真正把语言理解、视觉定位、路径规划和机器人执行打通成一条链。

### GridMM 精简版

GridMM 研究的是长程 VLN 里的历史环境表示问题。相比把历史压进 RNN hidden state，或者只用稀疏拓扑图，它提出维护一个可增长的俯视网格记忆，让历史观测持续写入地图，再把全局地图和局部候选信息一起用于动作决策。它的创新点在于把“地图结构本身”变成方法设计重点。我在复现时重点关注 global/local memory、位置特征、scoring 路径和 STOP 机制，并进一步思考把这类地图记忆思想往无人机 2.5D AirNav 场景迁移。
