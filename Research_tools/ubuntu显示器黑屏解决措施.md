# Ubuntu 外接显示器 30 秒自救清单

## 现象

**系统能检测到第二块屏，但外接屏黑屏 / 不能正常显示**

---

## 第一优先级：先在设置里改

打开：

**设置 → 显示**

按这个顺序试：

1. 把显示模式改成 **扩展显示**
2. 外接屏分辨率先设成 **1920×1080**
3. 刷新率先设成 **60Hz**
4. 缩放先设成 **100%**
5. 点“应用”
6. 不行就先切成 **镜像**
7. 应用后再切回 **扩展**

> 很多时候只是输出配置没真正生效。

---

## 第二优先级：如果你是 NVIDIA 机器

先看当前会话类型：

```bash
echo $XDG_SESSION_TYPE
```

如果输出是：

```bash
wayland
```

而且你又是 **NVIDIA 独显/混合显卡笔记本**，优先怀疑这个。

### 处理方法

退出登录，在登录界面：

* 选用户
* 点右下角齿轮
* 选择 **Ubuntu on Xorg**
* 登录后再试外接屏

> 对 NVIDIA 外接屏问题，这一步命中率很高。

---

## 第三优先级：检查显卡驱动

```bash
nvidia-smi
```

如果这条命令报错，说明 NVIDIA 驱动可能没正常工作。
去这里检查：

**软件和更新 → 附加驱动**

看有没有推荐驱动没启用，或者驱动状态异常。

---

## 第四优先级：命令行强制启用外接屏

先执行：

```bash
xrandr
```

看外接屏是不是 `connected`。

如果是，比如外接屏叫 `HDMI-1`，笔记本内屏叫 `eDP-1`，就试：

```bash
xrandr --output HDMI-1 --off
xrandr --output HDMI-1 --mode 1920x1080 --rate 60 --auto --right-of eDP-1
```

> 名字要按你机器实际输出改。

如果你以后再遇到，先看 `xrandr` 里的接口名，不要死记 `HDMI-1`。

---

## 第五优先级：排除线材 / 转接头 / 扩展坞

如果系统能识别显示器，但就是黑屏，也很可能是这些：

* HDMI 线有问题
* Type-C 转 HDMI 头不稳定
* 扩展坞兼容性差
* 显示器输入源选错
* 高刷链路不稳定

优先这样排：

1. 换线
2. 换接口
3. 不走扩展坞，直接连
4. 显示器菜单里手动切输入源
5. 先别开高刷

---

# 以后遇到同类问题的最短流程

## 30 秒版

1. 设置里改成 **1080p + 60Hz**
2. 镜像一次，再切回扩展
3. `echo $XDG_SESSION_TYPE`
4. 如果是 **Wayland + NVIDIA**，切 **Ubuntu on Xorg**
5. 还不行就 `xrandr` 看接口状态
6. 再换线 / 换口 / 绕过扩展坞

---

# 最常用的 3 条命令

```bash
echo $XDG_SESSION_TYPE
nvidia-smi
xrandr
```

你以后只要把这三条的输出看一眼，基本就能判断大方向：

* `wayland`：先怀疑会话兼容
* `nvidia-smi` 报错：先怀疑驱动
* `xrandr` 里 `connected` 但黑屏：先怀疑输出模式 / 刷新率 / 线材

---

**Ubuntu 外接屏“检测到了但黑屏”，优先排：1080p 60Hz → 镜像/扩展切换 → Wayland 改 Xorg → NVIDIA 驱动 → 线材/扩展坞。**
